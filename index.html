Untuk developer yang meragukan diri sendiri, apakah kamu cukup baik?

Apakah ada yang terdengar familiar?

kamu terus-menerus khawatir bahwa kamu tidak cukup baik.

Lihatlah para developer di sekitar kamu. Mereka tampaknya memiliki segalanya bersama. Kode mereka lebih bersih dari milik kamu. Mereka melakukan debug jauh lebih cepat daripada kamu. Mereka hanya melirik dokumentasi sebelum mengetahui apa yang mereka butuhkan. Mereka percaya diri. Mereka menikmati coding.

Tapi kamu khawatir bahwa kamu tidak cukup tahu tentang kerangka kerja(framework) dan bahasa yang kamu gunakan. kamu khawatir mengenalkan bug. kamu khawatir tentang mengejar teknologi terpanas. kamu khawatir bahwa kamu tidak cukup berbakat. Hal-hal tidak datang secara alami kepada kamu. kamu berpikir untuk diri sendiri: “Mungkin aku tidak dimaksudkan untuk menjadi seorang developer.”

Jika salah satu dari pikiran yang meragukan ini terdengar akrab bagi kamu, kamu tidak sendirian. Googling saja “keraguan diri seorang developer”, dan kamu akan menemukan lebih dari 2 juta hasil dalam waktu kurang dari satu detik.

Pelajaran # 1: Mempelajari pemrograman sama persis dengan membangun otot

Untuk belajar bermain sepak bola, kamu harus pergi ke lapangan dan bermain. Untuk membangun otot yang lebih kuat, kamu perlu pergi ke gym dan mengangkat beban tersebut. Untuk belajar pemrograman, kamu perlu duduk dan ngoding.

Setelah mengangkat beban, kamu akan merasa sakit dan lelah pada hari berikutnya. Ketika terjebak pada masalah pemrograman, otak kamu akan sakit dan kamu akan frustrasi.

Kamu tidak akan menyerah bekerja hanya karena merasa sakit. kamu tahu itu bagian dari proses. Tetapi ketika kamu terjebak saat pemrograman, kamu mungkin mulai meragukan diri sendiri, bertanya-tanya apakah kamu cukup berbakat.
Karena:

  1. kamu berpikir terjebak adalah kesalahan kamu,

  2. orang lain membuatnya tampak mudah.

    MARI KITA BAHAS

1. Terjebak adalah bagian dari proses.

Mungkin tidak ada yang memberitahumu, tapi itu benar-benar normal untuk terjebak selama pemrograman. Itu terjadi bahkan untuk para developer senior. Bagian besar dari pemrograman adalah tentang mencari tahu mengapa apa yang kamu harapkan terjadi namun tidak terjadi. Jika kamu setuju bahwa mengharapkan seorang anak untuk dapat berjalan saat pertama kali dia berdiri itu tidak masuk akal, mengapa kamu mengharapkan draf pertama kode kamu berjalan sempurna tanpa bug?

Jika menemukan error yang membuatmu frustsi. Bertahanlah di sana. Gali masalah ini. Tanya Google. Tanya teman. Jalan-jalan. kamu akhirnya akan menyelesaikannya dan melanjutkan. Dan ketika kamu mendapatkan lebih banyak pengalaman, kamu secara bertahap akan terjebak lebih sedikit.

Lain kali kamu terjebak, jangan menyulitkan diri kamu. Ingatkan diri kamu sendiri bukan salah kamu. Itu normal. Itu bagian dari proses.

PS - Menyiapkan lingkungan dev mungkin bagian yang paling sulit dari pemrograman.

Sangat umum untuk terjebak dan menghabiskan berhari-hari untuk membangun lingkungan developeran. Ini adalah momen yang bisa menghancurkan jiwa kamu. kamu mencoba mengikuti petunjuk langkah demi langkah. Namun kesalahan acak terus muncul di setiap langkah. Instruksi membuatnya tampak mudah, tetapi tidak. Bukan salahmu terjebak. 99,99% orang yang membaca tutorial itu macet. Sisanya 0,01% adalah yang beruntung yang kebetulan memiliki pengaturan sistem operasi yang sama persis dengan instruktur.

Sangat diakungkan bahwa langkah pertama untuk belajar program juga merupakan bagian yang paling membuat frustrasi dan menghancurkan jiwa. Saat kamu melihat kesalahan yang tidak dapat kamu pahami, cukup salin pesan error yang tepat dan Googling itu. (Menempatkan tkamu kutip di sekitar pesan kesalahan atau hanya mencari bagian pertama pesan mungkin membantu.) Coba solusi yang berbeda sampai kamu menemukan yang berhasil. Dan lanjutkan.

2. Kami bingung “ secara genetis pintar” dengan tampak pintar seseorang telah menghabiskan banyak waktu untuk subjek terkait”

Jika kamu sedang berjuang sementara orang-orang di sekitar kamu sangat hebat, jangan berpikir itu karena kamu tidak cukup berbakat. Kemungkinan besar, mereka hanya memiliki lebih banyak pemaparan terhadap subjek, atau subjek serupa, daripada kamu. Otak mereka telah membangun “otot” untuk menangani subjek jenis ini. Setelah kamu melewati perjuangan kamu saat ini, kamu akan tampak cerdas dan mudah juga.

Ini semua tentang repetisi dan set.

kamu membangun otot dengan melatihnya. Semakin banyak usaha yang kamu lakukan, semakin lama kamu berlatih, kamu semakin kuat. Belajar pemrograman memiliki proses yang sama persis. Semakin banyak waktu yang kamu habiskan untuk coding, semakin mahir kamu. kamu menuai apa yang kamu tabur.

Pelajaran # 2: Fokus pada keahlian kamu, bukan kamu

Berhenti memikirkan seberapa baik atau seberapa buruk programmer kamu. Jika kamu berada di bawah 5% dari tim kamu, kamu ingin menjadi lebih baik. Jika kamu berada di 5% teratas, kamu masih ingin menjadi lebih baik. Tidak peduli seberapa baik atau buruknya kamu, selalu ada orang di depan kamu dan orang-orang di belakang kamu. Jadi mengapa membuang waktu memikirkannya?

Sebaliknya, fokuslah pada keahlian kamu. Berhenti bertanya seberapa baik kamu. Tanyakan seberapa bagus kode kamu. Bagaimana kamu bisa membuatnya lebih baik, lebih cepat, lebih mudah dibaca? Kelilingi diri kamu dengan para insinyur yang kamu kagumi. Baca kode mereka. Apa bedanya dengan kamu? Apa trade-off yang mereka buat?

Jika kamu sudah bekerja sebagai insinyur perangkat lunak, seseorang membayar kamu untuk menulis kode. kamu harus menjeda dan merenungkannya, karena itu sangat berarti: seseorang percaya pada keterampilan pengkodean kamu atau melihat potensi kamu. Satu-satunya cara untuk mewujudkan potensi itu adalah fokus pada kerajinan kamu dan meningkatkannya tanpa henti.

Item Tindakan: Mulailah mencatat kemenangan kecil
Kapan terakhir kali kamu terjebak dan tidak yakin apakah kamu bisa menyelesaikan masalah? Bagaimana kamu akhirnya menyelesaikannya?

Buka Google Dokumen baru dan catat.

Lain kali jika kamu meragukan diri sendiri, lihatlah dokumentasi dan ingatkan diri kamu: jika kamu bisa menyelesaikan masalah terakhir itu, kamu akan menyelesaikan masalah ini juga.

Gunakan dokumen untuk melacak masalah menantang yang kamu atasi dan apa yang kamu pelajari dari mereka. Selalu, selalu rayakan kemenangan kecil ini!

Pengalaman Belajar Bahasa Go lang

Halloo people, kali ini frda akan membagikan
pengalaman bagaimana mengenal belajar bahasa baru yang lagi booming

Sebelumnya pernah denger ada bahasa baru yang cukup booming, frda bertanya-tanya mengenai bahasa baru ini, temen-temen yang belum tau bahasa Go lang itu apa, mungkin pertanyaannya bakalan sama kayak frda.

1. Apasih Go/Go lang itu?

Awalnya frda beranggapan kalo Go itu adalah bahasanya dan Go lang adalah frameworknya, jadi kayak berpasangan gitu. Tapi itu salah total bung hehehe.
Sebenernya bahasa Go dan Go lang itu sama aja. nama asli bahasanya adalah GO, tapi karena terlalu umum jadi ditambahin lang dibelakangnya sehingga sekarang terkenal dengan nama Go lang/Golang.

2. Kenapa Se-booming ini di kalangan programmer?

Frda bilang se-booming ini karena setiap ada event mengenai programming selalu ada sentilan bahasa Go, orang - orang ngomongin bahasa Go ini.
Ternyata faktanya adalah :

  • Bahasa Go adalah bahasa yang paling cepat eksekusinya dibandingkan dengan c#, javascript,dotNet dan lain -lainnya. Go lang ini eksekusinya/compilenya diibaratkan langsung ngomong ke komputer ga lewat interface. Hal ini bisa dibuktikan dan di tes dengan namanya benchmark, yang frda ketahui benchmark itu fitur di visual studio code yang berbentuk fungsi untuk membandingkan kecepatan eksekusi beberapa bahasa. Syntaxnya seperti ini go test -bench

  • Error Checking nya mantap, bisa dibilang Go lang ini pemberitahu yang baik, selalu ngingetin kalo ada variabel yang gak kepake(UnUsed) dia langsung error.

  • Cross Compiling, Go lang itu executable nya file binary

  • Garbage Collection, Go lang bakal me-manage memorinya secara otomatis

  • Scalability nya mendukung microservices dan modular

  • Simplicity katanya simpel

  • Syntaxnya agak berbeda dari yang lain, contohnya for each dalam Go:

for index, element := range someSlice {
    // index is the index where we are
    // element is the element from someSlice for where we are
}
If you don't care about the index, you can use _:

for _, element := range someSlice {
    // element is the element from someSlice for where we are
}
  • Bahasa Go ini lebih cocok dengan framework Echo karena kecepatan eksekusinya. Framework Echo sendiri merupakan framework Go yang High performance, extensible, dan minimalist.

3. Bagaimana Testing dalam Go?

Sebenernya ini pertanyaan yang belum kesampaian tapi tutorku memeperkenalkannya. Dalam Golang ada namanya TEST DRIVEN DEVELOPMENT atau disingkat TDD.
TDD adalah kita membuat test nya dulu untuk fungsi apa saja yang nanti akan digunakan, baru bikin kodingannya, atau bisa berbarengan satu - satu program/kodingan di buat lalu bikin test nya. Sehingga nanti kalo kodingannya banyak dan mau nambah fitur dalam aplikasi kita kita bisa tau dengan mudah errornya dimana dan lebih baik menambah fitur dimana.

Web Dev Trends 2018

Trend # 1: Pahami Dasar-dasarnya

Menjadi hal terpenting yang harus Anda ketahui sebagai pengembang web pada tahun 2018!
kita memiliki lebih banyak teknologi untuk pengembangan web, penting untuk kita memahami apa yang ada di dalamnya.

Trend # 2: Menyelam ke Node.js

Node.js adalah bahasa yang menawarkan kinerja hebat. Ini menampilkan ekosistem yang dinamis dan aktif dengan banyak paket pihak ketiga yang dapat Anda pasang ke aplikasi Anda.

Trend # 3: Jelajahi React, Angular dan Vue.js (Kerangka Pengarah)

Framework frontend yang populer di tahun 2018 adalahAngular, React dan Vue.js.
Kerangka kerja ini memungkinkan pengguna mengontrol di aplikasi web melalui JavaScript. Sehingga dapat membuat ulang bagian halaman (atau keseluruhan halaman) setelah dimuat. Ini menghemat pemuatan ulang halaman yang tidak perlu dan meminta siklus tanggapan <-> dengan server.

Trend # 4: Komponen Web & Komponen Kompiler

Sampai akhir 2017, dukungan browser adalah masalah yang lebih besar namun pada awal 2018, ini terlihat lumayan. Empat API Web yang membentuk komponen web - Unsur Khusus, DOM Shadow, Template HTML dan Impor HTML - didukung oleh semua browser utama.

Dengan elemen khusus dapat membuat elemen HTML Anda sendiri (misalnya ). Unsur-unsur ini berisi semua kode yang dibutuhkan untuk membawa sesuatu ke layar.

Trend # 5: Gunakan Generator Halaman Statis

Generator Halaman Statis seperti Jekyll, Hugo atau proyek seperti Nuxt.js untuk Vue.js (ini terinspirasi oleh Next.js for React) menyajikan versi halaman web yang telah ditentukan sebelumnya tanpa secara dinamis melakukan pra-render di beberapa server.

Trend # 6: Aplikasi Web Progresif (PWA)

Progressive Web Apps (PWA) adalah teknologi yang memungkinka kami membuat aplikasi web yang berperilaku sangat mirip dengan aplikasi seluler asli.
File manifest.json di sisi lain memberikan informasi meta yang berguna tentang aplikasi Anda ke browser, yang memungkinkan aplikasi dipasang di halaman depan.

Aplikasi bisa dipakai offline (misalnya, bagian dari aplikasi web yang berfungsi tanpa sambungan aktif - setelah aplikasi web dimuat setidaknya satu kali), kemampuan terinstal (misalnya, aplikasi web dapat ditambahkan ke layar utama perangkat) dan penggunaan perangkat asli fitur (seperti mengakses kamera perangkat).

Trend # 7: Bangunlah Aplikasi Tanpa Server

Serverless dapat dikatakan minim dalam mengelola dan memperbarui server, sehingga kapasitas server meningkat drastis dan Anda tetap hanya membayar untuk apa yang Anda butuhkan.
Cara bekerjanya adalah :

  • menulis kode Anda yang harus dijalankan sesuai permintaan
  • mengunggahnya ke layanan tanpa server(misalnya AWS Lambda)
  • menambahkan beberapa layanan tambahan (misalnya, AWS API Gateway untuk membangun REST tanpa server API)
  • menerapkan aset statis ke beberapa host statis seperti AWS S3.

Trend #8: Artificial Intelligence dan Bots

AI TIDAK terbatas pada mobil self-driving, hat bots di laman membantu pengguna memahami konten, alat pengenalan gambar dan pengoptimalan.
Ada banyak use-cases untuk komponen AI-driven di halaman web. Belajarlah Python, Machine Learning, Big Data, Deep Learning dan membangun alat AI sendiri.

Dan kita akan melihat banyak alat seperti ini muncul (dan menjadi lebih baik) di tahun 2018.

GIT ALA - ALA

Hai, ini rangkuman mengenai kode
atau perintah menggunakan Git tanpa ribet

1. Buat Repositori Baru (create a new repository)

Perintah untuk membuat repositori git baru:

  • get init

2. Periksa Repositori (checkout a repository)

Perintah untuk membuat salinan kerja dari repositori lokal, dan untuk remote server gunakan perintah ini:

  • git clone / path / to / repository
  • git clone username @ host: / path / to / repository

3. Alur-Kerja (workflow)

Repositori lokal terdiri dari Working Directory(untuk menyimpan filenya),Index(pengolah datanya), dan HEAD(commit terakhirnya)

4. Tambah dan commit (add & commit)

Untuk menambahkan/update ke index:

  • git add <filename>
  • git add *

Untuk memberi commit apa yang sudah di rubah diusahakan
commit messagenya informatif perintahnya:
git commit -m "Commit message"

5. Mengirim perubahan (pushing changes)

Perintah untuk me-remote mengirim perubahan:

  • git push origin master

Perintah untuk dihubungkan ke server jarak jauh
kalo repositori nya belum dikloning:

  • git remote add origin <server>

6. Percabangan (branching)

Dalam membuat repositori , Branch master adalah branch utama dan ia bawaan, kita bisa membuat branch lain misalnya feature_x.

  • git checkout -b feature_x Untuk masuk ke branch feature_x
  • git checkout master Untuk masuk ke branch master
  • git push origin <cabang> Untuk mengirim remote repositori

7. Perbarui & Gabung (update & merge)

Untuk update local repositori dengan commit terbaru gunakan perintah:

  • git pull
    Untuk mengambil dan menggabungkan perubahan dengan remote:
  • git merge <branch>
    Untuk menggabungkan cabang lain ke cabang aktif:
  • git add <filename>
    Untuk melakukan pratinjau menggunakan:
  • git diff <source_branch> <target_branch>

8. Menandai (tagging)

Buat tag dengan nama 1.0.0 dengan perintah:

  • git tag 1.0.0 1b2e1d63ff

1b2e1d63ff adalah 10 karakter pertama dari identitas commit yang di referensikan ke tag.

9. Log (log)

Perintah untuk melihat riwayat repositori

  • git log
    Perintah untuk melihat komit
  • git log --author=bob
    Perintah untuk melihat compressed log dalam commit 1 baris
  • git log --pretty=oneline
    Perintah untuk melihat ASCII art tree di semua branch yang didekorasi dengan nama tag dan branchnya
  • git log --graph --oneline --decorate --all
    Perintah untuk melihat file yang sudah diubah
  • git log --name-status
    Perintah untuk melihat lebih banyak parameter dalam git log
  • git log --help

10. Mengembalikan perubahan lokal (replace local changes)

Perintah untuk ke masa lalu jika ada yang ingin diubah
dengan konten terakhir dari HEAD :

  • git checkout -- <filename>

Perubahan dan berkas baru yang ditambahkan ke indeks
akan tetap tersimpan.

Perintah untuk menggagalkan perubahan dan komit lokal
seutuhnya, ambil riwayat terakhir dari server dan arahkan
ke cabang master lokal:

  • git fetch origin
  • git reset --hard origin/master

11. Petunjuk berguna (useful hints)

gitk

GUI git bawaan

git config color.ui true

Menggunakan output git penuh warna

git config format.pretty oneline

Menunjukkan log satu baris per komit

git add -i

Menambah dengan interaktif

Lebih lengkapnya lihat disini gitbook icip-icip
